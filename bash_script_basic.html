1 简介

1.1 hello world

#!/usr/bin/env bash
# 上面指定了使用什么解释器来解释此脚本

# A Simple Hello World Script

# 定义了一个变量
HELLO_WORLD="Hello World!"

# 定义了一个函数
sayHello() {
    # 使用了变量
    echo $HELLO_WORLD
}

# 调用函数
sayHello

# 正常退出
exit

如何执行? 第一种方式 bash <path-to-script> 。另一种给脚本加上可执行属性，直接用文件路径调用：

  • chmod +x <path-to-script>
  • <path-to-script>

上面这个例子中我们可以看到定义变量使用 var=value 即可，注意等号左右没有空格。 使用一个变量在前面加上美元符号即可。 另外我们还学到了，函数如何定义和调用。这是一个很重要的编程概念。 有了变量和函数，你已经能把脚本写的相当复杂了。 

foo() {}

foo {} # 括号可以省略，不过建议保留，增加可读性

1.2 参数处理

#!/usr/bin/env bash
s() {
    echo $# $@
    echo $1

    shift
    echo $# $@
    echo $1

    shift 2
    echo $# $@
    echo $1
}

ss() {
    for i in $@; do
        echo $i
    done
}

s "a" "b" "c" "d"
echo "==="
ss "a" "b" "c" "d"

输出： 

4 a b c d
a
3 b c d
b
1 d
d
===
a
b
c
d
  • $# 传入参数的个数
  • $n 引用第n个参数，第9个之后要使用 ${10} 这样的方式
  • $@ 传入的所有参数数组

将上面的最后一行改为下面的形式就可以从命令行接收参数了 

s $@

2.1 条件表达式 [[]] 3 , 7

用来测试一个条件的真值的表达式，用在 if, while, until 的判断分支中，注意括号内左右的空格必须要有

另外，条件表达式也可以与 && 和 || 一起用，写起来更加简洁 

[[ -f path_of_file ]] && echo $path_of_file
[[ -f path_of_file ]] || echo "Can not read file: $path_of_file"

2.3 $

常用的内置变量9

名称 解释
$0, $1… $0 脚本的执行路径，$n: 第 n 个参数
$# 参数个数
$* 所有参数合在一起的字符串
$@ 所有参数列表
$! 最后一个后台job的pid
$? 上一条命令的退出码
$$ 当前脚本的 pid

3 控制结构

#!/usr/bin/env bash

# seq n => 1 ... n
# seq m n => m ... n
# seq m s n => m ...(s)tep... n
for i in $(seq 10); do
    echo $i

    if [ $i -gt 5 ]; then  # Legacy Bourne
        echo "$i > 5"
    elif (( i > 3)); then  # POSIX
        echo "$i > 3"
    else
        echo "$i <=3"
    fi
done


declare -i count=0 # declare a integer variable
while (( count < 10 )); do
    echo "while count: $count"
    let count=count+1  # $ can be dropped in let statement
done

until (( count < 0 )); do
    echo "until count: $count"
    let count=count-1
done

case "$1" in
    start)
        echo "start stuff"
        ;;
    stop)
        echo "stop stuff"
        ;;
    restart)
        echo "restart stuff"
        ;;
    *)
        echo "Usage: $0 {start|stop|restart}"
        exit 1
esac

这里展示了基本的控制结构

  • $() 简单理解为限执行括号中的命令,拿到返回值后，再执行脚本，比较老的写法为 `cmd`
  • seq 是一条快速产生数字序列的命令
    • 单参数N， 1 到 N, N 个数字
    • 两参数M N, M 到 N， N-M+1 个数字
    • 三参数 M s N， M 到 N，步长为 s
  • (()) 算术上下文, 在这里可以省略变量前的美元符号，变量作为整数参与计算， let 与 (()) 的作用完全一样。 [] 是旧的写法， (()) 为 POSIX 写法10。

6 数字处理

bash 原生只支持整数，下面展示了计算 1-N 的整数和的递归解法13 

#!/usr/bin/env bash

sum() {
    # 定义了两个整型的局部变量
    local -i count=$1
    local -i result=$2

    if (( count == 0 )); then
        echo $result && exit
        # return $result # 这里并不是你想的那样
    fi

    result=$(( result + count ))
    count=$(( count - 1 ))
    sum $count $result
}

echo "Script path from cmd line: $0"
echo "Args: $@"

if (( $# < 1 )); then
    echo "Usage: $0 <count> ..."
    exit 1
elif (( $# > 1)); then
    echo "More then one args founded, we only take the first one as a integer"
fi

if [[ ! $1 =~ ^-?[0-9]+$ ]]; then
    echo "The first argument should be integer"
    exit 1
fi

result=`sum $1 0`
echo "Sum from 1 -> $1 is $result"

上面这个例子有几个点需要注意一下

  1. 检测第一个参数是否为数字时使用了正则表达式。此篇不做介绍

  2. sum 函数的返回值。 虽然 bash 提供了 return 语句，但是并不如其他语言那样能返回任何东西。 它只被设计为返回退出状态（一个整数）。上面输出 sum 结果的代码实际上时获取了 sum 函数中所有写到标准输出中的值。获取状态使用 #? ，并不是 = 号: 

    foo # call foo
statOfFoo=$? # get state code of last call

由于 bash 原生只支持整数，如果需要用到浮点数，就需要使用外部程序。一般使用 bc14 

#!/usr/bin/env bash

if (( $(bc <<< "3.0 > 1.5") )); then
    echo '3.0 > 1.5'
fi

a=$(bc -l <<< "1.0/3.0")
echo $a
printf '%.4f\n' $a

a=$(bc -l <<EOF
1 + 2 + 3 / 2.0
EOF
 )

echo $a
  • 浮点数的比较需要使用到 bc
  • bc -l 整数计算时不做截断
  • 使用 bc 会碰到计算结果小于1的小数不显示小数点左侧的0的情况，上面展示了使用 printf 命令处理的方式
  • 这里还展示了 here documents 的用法，可以读入复杂的表达式，而且不用考虑换行符和引号的问题
